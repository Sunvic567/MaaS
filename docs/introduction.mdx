## What is MemLayer?

MemLayer is a memory API for AI agents. It gives your agent persistent memory across conversations — so it remembers users, learns their preferences, and never asks the same question twice.

## The Problem MemLayer Solves

Every time a user starts a new conversation with your AI agent, it starts from zero.

It doesn't remember:
- Their name or location
- What they complained about last week
- Their communication preferences
- Anything they've ever told it before

You're either stuffing conversation history into the system prompt and hitting token limits — or your agent keeps asking the same questions over and over.

**MemLayer fixes that.**

## How It Works

<Steps>
  <Step title="Store">
    When your agent learns something about a user, call `store_memory()`. MemLayer embeds the text and saves it.
  </Step>
  <Step title="Retrieve">
    At the start of every session, call `get_context()`. MemLayer returns the most relevant memories before the user says a word.
  </Step>
  <Step title="Search">
    When the user asks something, call `search_memories()`. MemLayer returns memories ranked by semantic relevance, recency, and importance.
  </Step>
</Steps>

## Key Features

<AccordionGroup>
  <Accordion title="Semantic Search">
    Find the right memory for any query — not just exact keyword matches. MemLayer uses OpenAI embeddings to understand meaning, not just words. Ask "where does this user live?" and it finds "User is based in Lagos, Nigeria" even if the word "live" never appears in the memory.
  </Accordion>
  <Accordion title="Hybrid Scoring">
    Results are ranked by a combination of semantic relevance (70%), recency (20%), and importance (10%). A memory stored today ranks higher than an identical one from 6 months ago.
  </Accordion>
  <Accordion title="Duplicate Detection">
    MemLayer automatically detects near-identical memories before storing. No more storing "User is in Lagos" 50 times across 50 sessions.
  </Accordion>
  <Accordion title="TTL Expiry">
    Set memories to auto-expire after N days. Temporary session context disappears automatically — no manual cleanup needed.
  </Accordion>
  <Accordion title="BYOD Support">
    Enterprise clients can connect their own Supabase database. Your data never leaves your infrastructure.
  </Accordion>
</AccordionGroup>

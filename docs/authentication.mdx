## API Keys

Every request to the Remem API requires an API key in the `X-API-Key` header.

```bash
curl https://api.remem.online/memories/search \
  -H "X-API-Key: rm_live_xxx"
```

---

## Getting a Key

<Steps>
  <Step title="Sign up">
    Visit [remem.online](https://remem.online) and fill in your name and email.
  </Step>
  <Step title="Check your email">
    Your API key is sent immediately. Check your inbox (and spam folder).
  </Step>
  <Step title="Start building">
    Use the key in your requests. Free plan works immediately — no credit card needed.
  </Step>
</Steps>

---

## Key Format

Remem API keys always start with `rm_`: rm_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

<Warning>
  If your key doesn't start with `rm_` you'll get a `401 Invalid API key format` error immediately.
</Warning>

---

## Keeping Keys Safe

<AccordionGroup>
  <Accordion title="Use environment variables">
    Never hardcode your key in source code.

```python
    import os
    from remem import RememClient

    client = RememClient(api_key=os.getenv("REMEM_API_KEY"))
```

```bash
    # .env file
    REMEM_API_KEY=rm_live_xxx
```
  </Accordion>
  <Accordion title="Never commit to Git">
    Add `.env` to your `.gitignore`:
    # .gitignore
.env
.env.local

</Accordion>
  <Accordion title="Lost your key?">
    Contact support@remem.online with your registered email. We'll issue a new key and revoke the old one.
  </Accordion>
</AccordionGroup>

## Errors

| Code | Meaning | Fix |
|------|---------|-----|
| `401` | Missing or invalid key | Check your key starts with `rm_` |
| `403` | Account suspended | Contact support |
| `402` | Plan limit reached | Upgrade your plan |

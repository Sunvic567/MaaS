---
title: Search Memories
api: GET https://memlayer.online/memories/search
description: Semantic search — returns the most relevant memories for a query.
---

## Overview

Find memories using natural language. You don't need to match exact words — MemLayer understands meaning. Ask "where does this user live?" and it finds "User is based in Lagos, Nigeria" even though none of the words overlap.

Results are ranked by hybrid scoring: **70% semantic relevance + 20% recency + 10% importance.**

---

## Request

### Headers

| Name | Required | Description |
|------|----------|-------------|
| `X-API-Key` | Yes | Your MemLayer API key |

### Query Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `query` | string | Yes | — | Natural language search query. |
| `user_id` | string | Yes | — | Filter memories for this user. |
| `agent_id` | string | Yes | — | Filter memories for this agent. |
| `top_k` | integer | No | `5` | Number of results to return. Min 1, max 20. |
| `min_score` | float | No | `0.70` | Minimum hybrid score threshold. 0.0 returns everything. |
| `memory_type` | string | No | `null` | Filter by `episodic`, `semantic`, or `summary`. |

<CodeGroup>
```bash curl
curl "https://memlayer.online/memories/search?\
query=what+does+this+user+prefer&\
user_id=user_123&\
agent_id=support_bot&\
top_k=5&\
min_score=0.70" \
  -H "X-API-Key: ml_live_xxx"
```

```python Python SDK
memories = client.recall(
    "what does this user prefer?",
    user_id="user_123",
    agent_id="support_bot",
    top_k=5,
    min_score=0.70,
)

for m in memories:
    print(f"[{m.score:.3f}] {m.content}")
```

```python Python (httpx)
import httpx

resp = httpx.get(
```

```json
      "expires_at":   null
```

### Memory Object

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Memory UUID |
| `content` | string | The stored memory text |
| `user_id` | string | User this memory belongs to |
| `agent_id` | string | Agent that stored this memory |
| `memory_type` | string | `episodic`, `semantic`, or `summary` |
| `importance` | float | Importance score 0.0-1.0 |
| `score` | float | Final hybrid score for this query |
| `score_detail` | object | Breakdown of how the score was calculated |
| `metadata` | object | Any extra data stored with the memory |
| `created_at` | string | ISO timestamp when memory was stored |
| `last_accessed` | string | ISO timestamp of last retrieval |
| `expires_at` | string or null | ISO timestamp of expiry. `null` = permanent |

## Notes

<AccordionGroup>
  <Accordion title="Understanding score_detail">
    Use `score_detail` to debug why a memory ranked where it did:

```python
    for m in memories:
        print(f"Content:    {m.content}")
        print(f"Cosine:     {m.score_detail['cosine']}")
        print(f"Recency:    {m.score_detail['recency']}")
        print(f"Importance: {m.score_detail['importance']}")
        print(f"Final:      {m.score_detail['final']}")
```

    If the wrong memory is ranking first, check whether it has a much higher recency or importance score pulling it up.
  </Accordion>
  <Accordion title="Tuning min_score">
    `0.70` (default) — balanced. Good for most use cases.
    `0.85+` — strict. Only highly relevant memories.
    `0.0` — everything. Useful for debugging.

    Start with the default and adjust based on your results.
  </Accordion>
  <Accordion title="Empty results">
    If you get `total: 0`, check:
    - `user_id` and `agent_id` match exactly what you used when storing
    - Memories exist for this user (use `GET /memories` to list all)
    - `min_score` isn't set too high — try `0.0` to return everything
  </Accordion>
</AccordionGroup>

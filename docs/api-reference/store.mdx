---
title: Store Memory
api: POST https://memlayer.online/memories
description: Embed and store a memory for a user and agent pair.
---

## Overview

Call this whenever your agent learns something worth remembering about a user. MemLayer embeds the text using OpenAI and saves it to your account. Duplicate detection runs automatically — if a near-identical memory already exists, the store is skipped.

---

## Request

### Headers

| Name | Required | Description |
|------|----------|-------------|
| `X-API-Key` | Yes | Your MemLayer API key (`ml_live_xxx`) |
| `Content-Type` | Yes | `application/json` |

### Body

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `content` | string | Yes | — | The memory text to store. Max 10,000 characters. |
| `user_id` | string | Yes | — | Identifier for the end user. |
| `agent_id` | string | Yes | — | Identifier for the agent storing the memory. |
| `memory_type` | string | No | `episodic` | One of `episodic`, `semantic`, `summary`. |
| `importance` | float | No | `0.5` | Priority score from `0.0` (low) to `1.0` (high). |
| `metadata` | object | No | `{}` | Any extra data to store alongside the memory. |
| `ttl_days` | integer | No | `null` | Days until memory auto-expires. `null` = permanent. |

<CodeGroup>
```bash curl
curl -X POST https://memlayer.online/memories \
  -H "X-API-Key: ml_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "content":     "User prefers concise bullet points",
    "user_id":     "user_123",
    "agent_id":    "support_bot",
    "memory_type": "semantic",
    "importance":  0.8
  }'
```

```python Python SDK
from memlayer import MemLayerClient

client = MemLayerClient(api_key="ml_live_xxx")

result = client.remember(
    "User prefers concise bullet points",
    user_id="user_123",
    agent_id="support_bot",
    memory_type="semantic",
    importance=0.8,
)
print(result.id)
```

```python Python (httpx)
import httpx

resp = httpx.post(
    "https://memlayer.online/memories",
    headers={"X-API-Key": "ml_live_xxx"},
    json={
        "content":     "User prefers concise bullet points",
        "user_id":     "user_123",
        "agent_id":    "support_bot",
        "memory_type": "semantic",
        "importance":  0.8,
    }
)
print(resp.json())
```

## Response

### 201 — Success

```json
{
  "id":      "775263ee-73af-4416-9804-1f274048ae08",
  "message": "Memory stored successfully"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | UUID of the stored memory. Save this if you need to update or delete it later. |
| `message` | string | Confirmation message. |

### 409 — Duplicate

```json
{
  "detail": "Duplicate memory detected — skipped."
}
```

A near-identical memory already exists for this `user_id` and `agent_id`. Nothing was stored. This is safe to ignore.

### Error Responses

| Code | Meaning |
|------|---------|
| `401` | Invalid or missing API key |
| `402` | Plan memory limit reached |
| `409` | Duplicate memory — skipped |
| `422` | Validation error — check your request body |

## Notes

<AccordionGroup>
  <Accordion title="How duplicate detection works">
    Before storing, MemLayer embeds the new content and searches for similar memories. If any existing memory scores above 0.95 cosine similarity, the store is skipped. This prevents storing "User is in Lagos" 50 times across 50 sessions.
  </Accordion>
  <Accordion title="Choosing memory_type">
    Use `episodic` for things that happened ("User complained about delivery"). Use `semantic` for stable facts ("User is based in Lagos"). See [Memory Types](/concepts/memory-types) for details.
  </Accordion>
  <Accordion title="Setting importance">
    High importance (0.8-1.0) → always surfaced in retrieval. Low importance (0.1-0.4) → only retrieved when highly relevant. Default 0.5 is fine for most memories.
  </Accordion>
  <Accordion title="TTL for temporary context">
    Use `ttl_days` for session-specific context that should auto-expire. A memory with `ttl_days=1` disappears tomorrow — no manual cleanup needed.
  </Accordion>
</AccordionGroup>
